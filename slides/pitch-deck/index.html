Manual processes

Agencies generate large volumes of data but lack the analytics capability to act on it.

State-owned enterprises (utilities, transport) compound this: inefficient customer service, billing, and workforce management, with no systematic way to flag when something has gone wrong.

Method

Forecasting and anomaly detection, measured honestly

Holt-Winters triple exponential smoothing with weekly seasonality, backtested on a 14-day holdout for every forecasted series.

Z-score on forecast residuals flags time-series spikes. IQR peer comparison flags cross-sectional outliers: utility collection rates, budget execution.

Thresholds are calibrated to roughly 3 to 4.5 sigma specifically to avoid alert fatigue when scanning many parallel series at once.

Parallel compute

Queues for fan-out, one Durable Object for real state

Recompute fans out one job per region, agency, and metric partition across Cloudflare Queues: independent, stateless work, exactly what Queues are for.

Durable Objects are for coordination and shared state, not high fan-out computation. So a single Durable Object per job tracks real completion counts, backing an actual progress bar, not a decorative indicator.

AI transparency

The model narrates numbers. It never computes them.

Compute

Forecast and anomaly numbers are computed by the statistical pipeline and stored first.

Narrate

The model is given only those numbers and asked to explain them in Uzbek, Russian, or English.

Verify

Every number in the model's output is checked against the original grounding facts after generation.

Fall back

If a number does not match, the output is discarded and replaced with a deterministic template built from the same facts.

Result: the system stays accurate even if the AI API is unavailable, and every narrative ships with a "numbers behind this" panel.
